Perfecto. Vamos a diseñar **un agente IA conectado a tu base de datos PostgreSQL + Prisma + Ollama** que pueda **consultar y operar tu sistema de Acción Comunitaria** usando lenguaje natural.

La idea es que puedas preguntarle cosas como:

💬

* “¿Cuántas personas hay en la comunidad El Progreso?”
* “Muéstrame los afiliados de Juan Pérez”
* “Registra una persona nueva”
* “¿Cuántas personas hay en el distrito de La Chorrera?”

y el agente **automáticamente ejecuta consultas o acciones en la base de datos**.

---

# 1. Arquitectura del Agente IA

La arquitectura recomendada es esta:

```
Usuario
   │
Chat UI / API
   │
Agent Controller
   │
Ollama (LLM)
   │
Tool Router
   │
Skills (Funciones)
   │
Prisma ORM
   │
PostgreSQL
```

### Explicación

| Componente       | Función                    |
| ---------------- | -------------------------- |
| Frontend         | Chat con el usuario        |
| Agent Controller | Controla el flujo          |
| Ollama           | Comprende lenguaje natural |
| Tools            | Habilidades del agente     |
| Prisma           | Acceso a base de datos     |
| PostgreSQL       | Datos                      |

---

Para tu proyecto recomiendo empezar con estas.

---

# 3. Ejemplo de Skill

### Skill: crear persona

```javascript
async function crearPersona(data) {
  return await prisma.persons.create({
    data: {
      first_name: data.nombre,
      last_name: data.apellido,
      phone: data.telefono,
      community_id: data.community_id,
      leader_user_id: data.lider_id
    }
  });
}
```

# 4. Definir las Tools para el Agente

Ejemplo para el modelo:

```json
{
  "name": "crear_persona",
  "description": "Registrar una persona en el sistema",
  "parameters": {
    "type": "object",
    "properties": {
      "nombre": { "type": "string" },
      "apellido": { "type": "string" },
      "telefono": { "type": "string" },
      "community_id": { "type": "number" },
      "lider_id": { "type": "number" }
    }
  }
}
```

---

# 5. Flujo de Ejecución

### Usuario pregunta

```
Registra a Maria Lopez telefono 61234567 en comunidad 15
```

### LLM decide usar tool

Respuesta del modelo:

```json
{
 "tool": "crear_persona",
 "arguments": {
   "nombre": "Maria",
   "apellido": "Lopez",
   "telefono": "61234567",
   "community_id": 15
 }
}
```

### Backend ejecuta

```
crearPersona()
```

### Resultado

```
Persona registrada correctamente
```

# 6. Ejemplo con Ollama (Node.js)

Primero instalar:

```
npm install ollama
```

Ejemplo básico:

```javascript
import ollama from 'ollama'

const response = await ollama.chat({
  model: 'qwen2.5',
  messages: [
    {
      role: 'user',
      content: 'Cuántas personas hay en la comunidad 10'
    }
  ]
})

console.log(response.message.content)
```

---

# 7. Router de Tools

El backend debe detectar cuando el modelo llama una tool.

Ejemplo:

```javascript
if(response.tool === "personas_por_comunidad"){
   return personasPorComunidad(response.arguments.community_id)
}
```

---

# 8. Ejemplo de consulta con Prisma

```javascript
async function personasPorComunidad(communityId) {
  return await prisma.persons.count({
    where: {
      community_id: communityId
    }
  })
}
```

# 10. Seguridad del Agente

Debes limitar lo que el agente puede hacer.

Por ejemplo:

```
admin → todas las acciones
lider → solo sus afiliados
usuario → consultas básicas
```

Nunca permitir:

```
DELETE masivo
UPDATE global
```
